## Технологический стек

| Компонент | Решение |
|-----------|---------|
| Язык | Python 3.12+ |
| Telegram | aiogram 3.x, long polling |
| БД | PostgreSQL 16 |
| ORM | SQLAlchemy 2 (async) |
| Миграции | Alembic (применяются при старте бота) |
| FSM | MemoryStorage (только для регистрации) |
| Контейнеризация | Docker Compose (только PostgreSQL) |

**Удалённый репозиторий:** https://github.com/emilguseynov/pet-doodoo-bot.git

## Принятые архитектурные решения

| Вопрос | Решение |
|--------|---------|
| Часовой пояс | `Europe/Moscow` по умолчанию; команда `/timezone` — не реализована |
| Invite-код | 8 hex-символов в upper case (не UUID в строгом смысле, но 8 символов по ТЗ) |
| Миграции | Автоматически при старте (`src/db/migrations.py`) |
| Сессия БД | Одна транзакция на входящее обновление (`DatabaseMiddleware`) |
| FSM регистрации | `MemoryStorage` — достаточно для коротких сценариев |
| Команда Event Log | Запланирована как `/eventlog` (этап 6) |

## Как запустить локально

```bash
cd pet-poop-bot
cp .env.example .env          # вставить BOT_TOKEN от @BotFather
docker compose up -d          # PostgreSQL
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python -m src.main
```

Переменные окружения:

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Токен Telegram-бота |
| `DATABASE_URL` | `postgresql+asyncpg://petbot:petbot@localhost:5432/petbot` |
